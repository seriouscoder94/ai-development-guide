Content Creation Examples

Poor Prompt:

❌ "Write a blog post about technology"

Why it's poor: Too vague, no specific topic, audience, or format specified.

Effective Prompt:

✅ "Write a 1000-word blog post about the impact of artificial intelligence on remote work. Target audience: HR professionals and business managers. Include: - Current trends in AI-powered remote work tools - 3 specific case studies of successful implementations - Practical tips for implementing AI tools in remote teams - Potential challenges and solutions Tone: Professional but accessible. Format: Include headings, bullet points, and a conclusion with actionable takeaways."

Why it works: Specifies length, topic, audience, structure, and desired outcomes.

Technical Problem Solving

Poor Prompt:

❌ "Fix my React code"

Why it's poor: No code provided, no error message, no context about functionality.

Effective Prompt:

"I'm building a React component that fetches and displays user data. Here's my current code:

```jsx
const UserProfile = () => {
    const [userData, setUserData] = useState(null);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        fetch('/api/user')
            .then(res => res.json())
            .then(data => setUserData(data))
            .finally(() => setLoading(false));
    }, []);

    return (
        
{userData?.name || 'Loading...'}

    );
}
```

Issues I'm experiencing:
1. Component doesn't handle API errors
2. No loading state displayed
3. Need to add retry functionality

Environment:
- React 18.2.0
- Node.js 16.x
- Running in development mode

Can you help me improve this code with proper error handling, loading states, and retry logic?"

Why it works: Includes complete code, specific issues, environment details, and clear requirements.

Data Analysis Examples

Poor Prompt:

❌ "What does this data mean?"

Why it's poor: No context, no specific analysis goals, no data format specified.

Effective Prompt:

"I have a CSV dataset of customer transactions with the following columns:
- transaction_date
- customer_id
- product_id
- purchase_amount
- channel (online/in-store)

Please help me analyze this data to:
1. Identify customer segments based on:
   - Purchase frequency
   - Average transaction value
   - Preferred shopping channel

2. Detect patterns in:
   - Seasonal buying behavior
   - Product category preferences
   - Customer lifetime value

3. Generate visualizations for:
   - Monthly revenue trends
   - Customer segment distribution
   - Channel performance comparison

Format the results as:
- Summary statistics for each analysis
- Key insights and recommendations
- Suggested next steps for marketing team

Here's a sample of the data:
[data sample]"

Why it works: Specifies data structure, analysis goals, desired visualizations, and output format.
